The Prowl Programming Language

The Prowl Programming Language

Prowl has undergone a lot of changes recently! You can view the old tutorial in the "old" tab (which is implemented in the main branch of the repo), or you can read the new plans here!

Prowl is a statically typed, functional concatenative programming language based on the algebra of binary relations (functions with many outputs) and arrows (function-like objects). It is concatenative on the term level (permitting infix and bindings), higher-order (supporting quotes), and aims to extend the concatenative paradigm beyond the simple Stack -> Stack model of Forth, Joy, Factor, and other previous stack languages. Prowl's goal is also to modernize the paradigm by adding ML-like syntax, type inference, and modules that make programming more productive and more abstract.

To begin with, Prowl uses a nondeterminism system to extend the functions used in most other languages to binary relations. Relations pick up certain nice properties compared to functions: in addition to supporting function-like composition, they are composable "sideways" too, where two relations of the same type can share their inputs and have their outputs add together, creating a new relation. This operation, which we will call alternation, is a type of sum that can be related to composition in the direction of functions - concatenation, which is a type of product. These have the properties you would expect a sum and product to share: the distributive property, annihilation, and so on. The possibilities from alternation are accessible to the programmer in the form of a lazy stream, with full list operations available and control over the time of execution. Choosing relations as the basis of our language gives us several other benefits: functions and lists are pervasively useful anyway, and there are rich algebraic features including surprising connections to regex that make the language highly expressive. The nondeterminism system is checked strongly by refinement types so as to stay manageable and to carry programmer intentions in types.

Secondly, Prowl uses Arrows (function-like data) extensively as another composeable abstraction. Arrows overlap with stars in many ways (sharing the same function composition), but also permit some new operations like function pairing, combining the inputs and outputs, and function choosing, the ability to use one of many functions. Arrows are able to encapulate a wide range of useful behaviors, including IO and state management, exceptions & control flow, parsing, and just computation in general.

Prowl takes a significant amount of inspiration from the ML family of languages, including static typing and type inference, let bindings, infix operators, algebraic data, and modules. Prowl implements the powerful MixML module system, and modular implicits for ad-hoc polymorphism.
