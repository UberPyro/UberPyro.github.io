The Prowl Programming Language

Stack Semantics

Let's begin our journey into Prowl by considering some simple examples of operations on the stack. Prowl at it's core is a stack-based language, which means that every expression is a function of the stack. Values are typically realized as functions that push to the stack, and functions in general consume their arguments from the stack, and then push to the stack. Programs written in this style are often terse and highly composable, which should become more clear moving forward.

Prowl has a calculator mode which allows it to evaluate programs without the need for a main function.

At the start of program execution, the stack is empty. In calculator mode, the contents of the stack is printed to the screen at the end of execution. For example: suppose our file is just a number. Input: 

5

Output: 

5

The integer literal 5 pushes the value 5 onto the stack.

What if there are multiple literals? 

7 8 9

Output: 

7
8
9

The contents of the stack is printed, final value last.

Arithmetic

What if we want to do useful work? Prowl supports several notations for carrying out math.

  • Binary operators by default are infix, like a + b.
    • 1 + 3 evaluates to 4.
    • 1 + 4 * 5 evaluates to 21, following order of operations.
    • (1 + 4) * 5 evaluates to 25, using (..) for grouping.
  • Binary operators can be sectioned to be evaluated in Reverse-Polish Notation.
    • (+) is fully sectioned, taking two inputs off the stack.
      • 2 3 (+) evaluates to 5.
      • 5 4 3 (-) (+) evaluates to 5 1 (+) and finally 6.
    • (+ a) is left-sectioned, taking one input off the stack.
      • 5 (- 1) evaluates to 4
      • 5 (- 1) (* 2) evaluates to 8.
    • (a +) is right-sectioned, taking one input off the stack.
      • 5 (1 -) evaluates to -4, equivalent to 1 - 5.
  • Concatenation is faster than all infix operators.
    • 5 4 (-) + 2 evaluates to 1 + 2 and then 3.
    • 5 4 (-) + 2 (+ 1) evaluates to 1 + 3, and then 4.
    • 5 4 + 2 is a type error, because 5 4 is not an integer!
      • Infix operators in concatenative languages can have some strange semantics. We will explore how exactly they are encoded into the language later.
