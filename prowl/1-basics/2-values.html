The Prowl Programming Language

Values

Where expressions can be thought of as functions between stacks, there are two different ways to think of values:

  • As the elements on the stack
  • As expressions that place a particular representative element on top of the stack.

You may find the first to be more intuitive when binding to identifers, and the second to be more intuitive when using identifiers or literals.

As Expressions

Prowl uses lexically scoped variables for convenience. as expressions are a simple binding construct as <pattern> -> <expression> that binds a pattern (in this case, an identifier) and is followed by an expression (in which bound identifiers are in scope). Note that identifiers must begin with a lowercase letter, and can contain caps, numbers, -, and _.

  • 5 as x -> x evaluates to 5, as 5 is popped from the stack and bound to x.
  • 4 5 as x -> 6 x evaluates to 4 6 5. The as expression does not touch the 4.
  • 5 as _ -> 2 evaluates to 2. _ typically indicates that a value is being thrown away.
  • 5 as x -> x + 1 evaluates to 6.
  • 5 - 2 as x -> x - 1 evaluates to 4 because concatenation is faster than subtraction.
  • 5 - 2 as x y -> x - y evaluates to 3. Both the 5 and 2 are popped and bound.

Value Definitions

Outside of our calculator mode, the top level consists of a series of definitions rather than an expression. Value definitions can bind identifiers.

  • val x = 5 binds 5 to x.
  • val x = 5 + 4 binds 9 to x.
  • val x y = 2 3 binds 2 to x and 3 to y.
  • val x y = 3 is a type error, because the stack created on the right is too small for the patttern.
  • val x = 3 4 is a type error, because the stack is too large for the pattern. 
val x = 4
val y = x + 1

Is an example of how definitions can be sequenced.

Specifications

Different values can have different types associated with them. 

/-lit      type -/
5       /- int -/
4.6     /- float -/
'a'     /- char -/
"hello" /- str -/

You can assert that a value has a particular type with a specification. 

spec x : int
val x = 5

spec y : float
val y = 6.7

spec z : char
spec w : str
val z w = 'a' "hello"

Type Annotations

You can also declare the type inline by annotating the identifer. Specifications and annotations are not necessary, as Prowl has full type inference, though it may be useful for debugging. 

val x : int = 5
val z w : char str = 'a' "hello"
