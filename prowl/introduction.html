The Prowl Programming Language

The Prowl Programming Language

Prowl has undergone a lot of changes recently! You can view the old tutorial in the "old" tab (which is implemented in the main branch of the repo), or you can read the new plans here!

Prowl is a statically typed, functional concatenative programming language that aims to evolve the concatenative paradigm and introduce some cool new ideas along the way.

Prowl has 3 major influences that you should know about:

  • Kitten
  • Vinegar
  • Oniguruma Regex

The last one may come as a surprise, but to understand it, it helps to understand Vinegar first.

Concatenative languages are traditionally composed of sequences of words, where every word is a function Stack -> Stack. Rather than having arguments and a return value, everything has an input stack and an output stack, and everything is a function, even literals. If you haven’t checked out the concatenative paradigm yet, I would recommend doing so first! This paradigm is interesting because it allows you to write highly factorable code using short memorable words—flexible and composable. But a question remains: what can we do to extend this paradigm? Is Stack -> Stack the best we can do, or can we do better?

Vinegar is an esolang (though Prowl tries to not be one) where every operation can fail. Every function has type Stack -> Maybe Stack, and in case a sequence fails, it can be caught with an alternation operator |. This is a simple extension to the language, but it is powerful—we have a new way to reason about error handling and option types that frees us from the need to use combinators, which are often difficult to understand. This is a great first step in breaking the Stack -> Stack tradition.

Secondly, let’s talk about regex—the most curious influence we have listed. From above, we can see that we have two important associative operators: concatenation, denoted by (space), and alternation, denoted by |. The astute among you may have realized something very similar to a product and sum relationship: concatenation, a sort of product, distributes over alternation. Additionally, concatenation with a function that always fails meets the annihilating property, guaranteeing failure. Lastly, we spot idempotency in |, alternating the same thing twice either fails twice or succeeds the first time, solidifying concatenation and alternation as a Kleene algebra, the mathematical basis of regex. This empowers us to think of concatenative programs like regex.

  • Words are like characters
  • Concatenation f g h is like sequencing abc
  • Alternation f | g is alternation a|b

Now we can now mix in and reason about what the other regex operators mean in our language, as everything can be encoding in what we already have. For the following, imagine we employ a longest-match first strategy instead of using the first match:

  • f* can be represented as identity | f | f f | f f f | … corresponding to while
  • f+ is f | f f | f f f | …, do while
  • f? is identity | f, try or do nothing, a common idiom.
  • f{x} using the Oniguruma notation is f f f… x times, acting like a for loop

I hope this motivates what Prowl is trying to do. Prowl is like this, but to the highest level. Prowl uses a nondeterminism system, checked by refinement types, to extend the idea of possibilities generated by regex to lazy streams, to offer powerful and expressive forms of control flow, error handling, and list processing. Prowl uses stack-based semantics supplemented with a costack, a stack hammered out into a long chain of choices rather than a long chain of pairs, to create a form of error handling that's safe, comfortable, and unlike anything you have seen before. Prowl abstracts the idea of functions to function-likes, or arrows, to improve the composability of data and safely handle IO, state, and much more, and Prowl abstract the idea of relations (many-valued functions) to what I call stars, so that regex operations are available on a variety of operations and data.

We are not done yet: Prowl additionally strives to make several syntactic improvements to the concatenative paradigm to make it natural and easy to program in. Here, we take great inspiration from Kitten and from the ML family of languages at large. We introduce lexically scoped variables to ease the passing of data, and several kinds of binding constructs to make this natural in expressions and definitions. We forgo concatenative purity and introduce infix operators as a more natural way to reason about math and varies forms of composition. We have syntactical elements that correspond to different forms of control flow for easy of use and readability, ultimately fitting code into a terse shell. Finally, we provide an advanced, modern module system based on MixML for organization, interfaces, and powerful abstractions, and augment this with a modular implicits system for ad-hoc polymorphism.

Prowl is ambitious, unusual, creative, and quirky. I don’t expect many of the ideas to be a slam dunk, but I hope that the few parts that do turn out useful can inspire more mainstream languages, or for it to be cannibalized and done better. Most of all, I hope I can (re)ignite the belief that there are plenty of really good, really interesting ideas out there to be discovered, that programming languages can be fascinating, and that the field is brutally underexplored. Exploring new paradigms with new syntax and new ideas can be very rewarding in our arid age of old, established languages, and some similar, new languages, that mostly look the same and work about the same due to a history and a culture that is hard to overturn.
